### 題目意思
* 情境：給定 $n$ 種不同面額的硬幣，以及一個目標總金額 $x$
* 已知條件：
  * 硬幣數量無限，同一種硬幣可以重複拿取
  * 組合不計順序：這題求的是「組合數」（Combinations），意思是**順序不同視為同一種方法**（例如：`2+2+5` 和 `5+2+2` 算作同一種）。
  * 答案可能很大，輸出時必須對 $10^9+7$ 取餘數（Modulo）
* 目標：求出湊出總金額 $x$ 共有「幾種」不重複的組合方式？

### 核心概念
* 思維轉換：經典的「完全背包問題」變形。為了確保算出來的是「組合數」而非「排列數」，**迴圈的順序是最大的關鍵**
* 使用動態規劃（DP）來解
  * 狀態定義：`dp[j]` 代表「湊出金額 `j` 的組合方法總數」。陣列大小開 `x + 1`，型態用 `long long`
  * 初始條件：`dp[0] = 1`。湊出金額 0 的方法只有 1 種（就是什麼硬幣都不選），這是所有推導的起點。
  * 迴圈順序：
    * **外層迴圈跑「硬幣種類」**（`for i from 0 to n-1`）
    * **內層迴圈跑「目標金額」**（`for j from v[i] to x`）
    * *原理解釋*：這樣代表我們「先徹底考慮完第一種硬幣，再去考慮第二種硬幣」。因此硬幣加入的順序一定是被固定好的，完美避開了 `2+5` 和 `5+2` 被重複計算的問題
  * 狀態轉移方程：
    * `dp[j] = (dp[j] + dp[j - v[i]]) % MOD`
    * 給定當前硬幣 `v[i]`，湊到金額 `j` 的方法數，等於「原本湊到 `j` 的方法數」加上「湊到 `j - v[i]` 的方法數」
  * 實作細節與優化：
    * **內層迴圈起點優化**：不要從 `j = 1` 開始跑再用 `if` 判斷，直接讓金額 `j` 從當前硬幣面額 `v[i]` 開始起跑（`j = v[i]`）。這樣確保 `j - v[i]` 永遠 $\ge 0$，寫法更乾淨且執行更快，同時避免誤觸 `for` 迴圈提早 `break` 的陷阱
    * **步步取模**：因為是累加，每次加完都要立刻 `% 1000000007`，確保陣列內每個數字隨時都在安全範圍內
