### 題目意思
* **情境**：書店有 $n$ 本書，每本書都有對應的「價格」與「頁數」。你身上帶了 $x$ 元的預算。
* **規則**：每本書**最多只能買一次**（買或不買）。
* **目標**：在總花費不超過 $x$ 元的前提下，求你能買到的「最多總頁數」。

### 核心概念
* **看破題型：0/1 背包問題**
  這題是標準的 0/1 背包問題（0/1 Knapsack Problem）。
  把「價格」視為物品的重量 / 體積，「頁數」視為物品的價值，「預算 $x$」視為背包的總容量。

* **狀態定義 (1D DP 陣列)**
  * `dp[j]` 代表「當花費剛好（或不超過） `j` 元時，所能獲得的最大總頁數」。
  * 初始狀態：`vector<ll> dp(x + 1, 0)`，一開始所有預算下的頁數都是 0。

* **狀態轉移方程式**
  * 決定要不要買第 `i` 本書時，比較「不買」與「買」的結果，取最大值：
  * `dp[j] = max(dp[j], dp[j - prices[i]] + pages[i])`

* **實作細節與致命陷阱 (迴圈方向)**
  要將原本的 2D DP 壓縮成 1D DP 並保證「每本書最多買一次」，迴圈的設計非常嚴格：
  1. **外層迴圈**：遍歷每一本書（`for i = 0 to n-1`）。
  2. **內層迴圈（極度重要）**：遍歷預算容量，且**必須從大到小逆向跑**（`for j = x down to prices[i]`）。
  * **為什麼要逆向？** 因為如果從小到大更新，當你更新大預算 `dp[10]` 去查小預算 `dp[5]` 時，`dp[5]` 可能已經買過當前這本書了，這樣會導致同一本書被重複購買（變成「無限背包問題」）。逆向更新能確保查到的歷史狀態都是「上一本書」留下的乾淨狀態。
