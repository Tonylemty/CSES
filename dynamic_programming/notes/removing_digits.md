### 題目意思
* 情境：給定一個整數 $n$。每一次操作，你可以選擇減去該數字目前的「其中一個位數」。
* 條件：例如當前數字是 27，你可以選擇減去 2 或是減去 7。
* 目標：求出把數字 $n$ 扣到完全變成 0，**最少**需要經過幾次操作（幾步）？

### 核心概念
* 思維轉換：這是經典題「求最少硬幣數（Minimizing Coins）」的精妙變形。不同於固定面額的硬幣，這裡能用的「硬幣」，是**隨著當前數字動態改變的位數**。
* 使用動態規劃（DP）來解
  * 狀態定義：`dp[i]` 代表「把數字 `i` 扣到變成 0 的最少步數」。因為要求最小值，陣列初始化時必須塞滿一個極大值（例如 `1e9`），讓真正的答案能覆蓋過去。
  * 初始條件：`dp[0] = 0`。數字 0 變成 0 只需要 0 步，這是所有推導的地基。
  * 狀態轉移方程：
    * 針對每一個數字 `i`，把它拆解出每一個位數 `digit`。
    * `dp[i] = min(dp[i], dp[i - digit] + 1)`
    * 原理：走到 `i` 的最少步數，就是去查看「扣掉 `digit` 之前的那個數字（`i - digit`）的最少步數」，再加上你決定減去 `digit` 的這 **1** 步。
  * 實作細節與效能優化：
    * **避開字串地雷**：絕對不要用 `to_string()`！頻繁地建立與銷毀字串物件會耗費大量記憶體與時間，在極限測資下很容易超時（TLE）。
    * **數學拆解法**：使用 `while (temp > 0)` 搭配 `temp % 10`（抓出最後一個位數）和 `temp / 10`（砍掉最後一個位數），讓 CPU 用最基礎的純數學指令瞬間算完。
    * **防呆機制**：如果抓出來的 `digit` 是 0，因為扣 0 數字不會變，對推進進度沒有幫助，加上 `if (digit != 0)` 判斷直接略過即可。
